# ユビキタス言語（Ubiquitous Language）概要

## 定義

ユビキタス言語とは、ドメイン駆動設計（DDD）における中核的な概念で、開発者とドメインエキスパート（業務専門家）が共通して使用する厳密な言語のことです。

> "Ubiquitous Language is the term Eric Evans uses in Domain Driven Design for the practice of building up a common, rigorous language between developers and users."
> — Martin Fowler

## 主要な特徴

### 1. 共通性
- 開発チーム全体（開発者、ドメインエキスパート、ビジネスステークホルダー、ユーザー）が共有する言語
- チームの会話、ドキュメント、コードで一貫して使用される
- 同じ用語が同じ意味を持つ（境界づけられたコンテキスト内で）

### 2. 厳密性
- 曖昧さを排除した明確な定義
- 業務プロセスを一意に識別
- 認識のズレを防ぐ

### 3. 発展性
- チームのドメイン理解の深まりに応じて進化
- 継続的な見直しと更新が必要
- 実践を通じて洗練される

## なぜユビキタス言語が重要なのか

### コミュニケーションの効率化
- 開発者とドメインエキスパート間の認識合わせが不要になる
- コミュニケーションエラーの削減
- スムーズなチーム内コミュニケーション

### ビジネス理解の促進
- コードを読むだけでビジネス要件が理解できる
- 業務プロセスの可視化
- 現行モデル構造の問題点の特定

### コード品質の向上
- ソースコードの変数名、メソッド名、クラス名にユビキタス言語を反映
- ビジネスドメインとコード構造の一致
- 技術者・非技術者双方が理解できるコード

## ユビキタス言語の範囲

ユビキタス言語は**境界づけられたコンテキスト（Bounded Context）**内で有効です。同じ用語でも異なるコンテキストでは異なる意味を持つ場合があります。

### 例
- 「顧客」という用語は、販売コンテキストとサポートコンテキストで異なる属性や振る舞いを持つ可能性がある
- 各コンテキストで明確に定義し、境界を意識することが重要

## DDDにおける位置づけ

ユビキタス言語は、DDDの戦略的設計と戦術的設計の両方を支える基盤となります：

- **戦略的設計**: 境界づけられたコンテキストの定義
- **戦術的設計**: エンティティ、値オブジェクト、集約などのモデリング

すべてのDDDパターンは、ユビキタス言語によって命名され、その意味が定義されます。
