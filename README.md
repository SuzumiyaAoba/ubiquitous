# ユビキタス言語システム (Ubiquitous Language System)

**ドメイン駆動設計（DDD）のためのユビキタス言語管理プラットフォーム**

## 🎯 プロジェクト概要

このプロジェクトは、ドメイン駆動設計を実践するチームが、ユビキタス言語を効果的に管理・共有・進化させるための専用システムです。開発チームとドメインエキスパートの間で共通の語彙を確立し、ドメインの理解を深め、より良いソフトウェア設計を実現することを目指します。

## 🤔 解決する課題

### 現状の問題点
- ユビキタス言語の用語が複数のドキュメントに散在し、最新の定義を見つけにくい
- 同じ概念に対して異なる用語が使われ、混乱が生じる
- 新規参加メンバーがドメイン用語を理解するのに時間がかかる
- コードとドキュメントで用語の不一致が発生する

### 目指す姿
- ユビキタス言語の一元管理と容易なアクセス
- チーム全体での共通理解の促進
- ドメイン知識の組織資産化
- 新規メンバーのオンボーディング時間を50%削減

## 👥 対象ユーザー

1. **DDDを実践する開発チーム** - アジャイル/スクラム開発、モデリング重視のチーム
2. **ドメインエキスパート/ビジネスアナリスト** - 業務知識を持つステークホルダー
3. **プロダクトオーナー/プロジェクトマネージャー** - チーム間のコミュニケーション促進役
4. **新規参加メンバー** - プロジェクトに新しく参加した開発者

## 📊 期待される成果

### 定量的目標
- 新規メンバーのオンボーディング期間: **2週間 → 1週間**
- 用語の認識齟齬によるバグ: **40%削減**
- ドメインエキスパートとの会議での確認時間: **30%削減**
- コードとドキュメントの用語一致率: **90%以上**

### 定性的目標
- チーム全体でのドメイン理解の深化
- ドメインエキスパートと開発者の認識ギャップ解消
- ユビキタス言語に基づいた一貫性のある設計
- ドメイン知識の組織資産化

## 🛠️ 技術構成

### モノレポ構造
このプロジェクトはTurborepoを使用したモノレポ構成です：

```
├── apps/
│   ├── api/          # Honoバックエンド API
│   └── web/          # Next.jsフロントエンド
└── packages/
    └── types/        # 共有TypeScript型定義
```

### 技術スタック
- **フロントエンド**: Next.js 14 (App Router), React 18, TypeScript
- **バックエンド**: Hono, TypeScript
- **データベース**: PostgreSQL
- **検索エンジン**: MeiliSearch
- **AI統合**: OpenAI API
- **ビルドツール**: Turborepo

### セットアップ方法

詳細は [PROJECT_SETUP.md](./PROJECT_SETUP.md) を参照してください。

```bash
# 依存関係のインストール
npm install

# 環境変数の設定
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env

# 共有型パッケージのビルド
npm run build --workspace=@ubiquitous/types

# 開発サーバーの起動
npm run dev
```
